- input/realtime_text_monitor.py

Observed from logs:
- Foreground window was recognized as OpusApp
- Focused/target control was:
  PaneControl class='_WwG' name='Microsoft Word 문서'
- But text candidates remained empty

Assistant tried multiple Word strategies:

1. UI Automation candidate extraction
- Did not produce real body text

2. Word-specific clipboard snapshot fallback
- Focus the document area
- Send Ctrl+A and Ctrl+C
- Read clipboard
- Restore original clipboard
- Still did not solve the issue in the old Word environment

3. COM-based fallback
- Added attempt to read:
  - ActiveDocument.Content.Text
  - Range().Text
  - Selection.Range.Text
- But COM ActiveObject access failed or did not meaningfully return text in this environment

Findings:
- Word 2007 may not expose automation data as modern versions do
- ActiveObject access raised:
  OSError [WinError -2147221021] 작업을 사용할 수 없습니다

Conclusion at this stage:
- Word on this machine is a poor target for continuing development
- Future work should move to another computer with:
  - newer Word
  - installed Hangul/HWP
